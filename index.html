Everyday Git

1. Get Started

First, set up your development environment. For this tutorial, you’ll need to install Visual Studio Code and create a folder named everyday-git where you will place all your files.

2. GIT

What is Git?

Git is a Version Control System (VCS). A version control system is software that tracks and manages changes to files over time. It allows developers and teams to keep a history of their project, enabling them to revisit earlier versions of their files, compare changes between versions, undo unwanted changes, and more.

In this blog, we will focus on Git and how to use it with GitHub, a popular platform for hosting Git repositories.

What Git does for us

  • Track changes across files.
  • Compare versions of a project.
  • Time travel back to older versions.
  • Revert to a previous version.
  • Collaborate and share changes.
  • Combine changes.

Brief History about Git

In 2005, while developing Linux, Linus Torvalds grew frustrated with the available version control systems (VCSs), as they were slow, proprietary, and often required payment. Torvalds sought a VCS that was fast and free, unlike the tools that were available at the time. On April 3rd, 2005, he began creating his own VCS, which would later become Git. Within just a few days, he had built the basic functionality. Today, Git is open source, supported and developed by a global community of programmers.

Version control systems play a crucial role in software development, especially when multiple developers are working on the same project. They ensure that all changes are tracked and that everyone is working with the most up-to-date version of the code.

There are several other version control systems like Git, such as:

  • Subversion (SVN)
  • CVS (Concurrent Versions System)
  • Mercurial

3. Installing Git

To check if Git is installed on your machine, open a terminal and run the following command:

git --version

If Git is not installed, follow the instructions for your OS below:

  • Windows: Download from the official Git website.
  • macOS: Run git --version in the terminal to install Git if it’s not already installed.
  • Linux: Run sudo apt install git for Ubuntu-based systems.

Confirm whether you have Git installed

  • Open command prompt in windows or terminal in mac/linux.
  • Run the command:
    • git --version
  • If a version number is returned, Git is installed on your system. If you receive any other output, Git is not installed.

4. Configuring Git

Now that Git is installed, it's time to configure basic information. We don't need to 'register an account' to use Git, however, we need to tell Git who we are. We do this by providing our username and email address.

Configuring your username

Configure your username using the following commands:

git config --global user.name "USERNAME"

Replace USERNAME with your preferred username, for example, if your username is kinya-njui:

git config --global user.name "kinya-njui"

Configuring your email address

Configure your email address using the following commands:

git config --global user.email "EMAIL_ADDRESS"

Replace EMAIL_ADDRESS with your preferred username, for example, if your username is kinyanjuicharlesm@gmail.com:

git config --global user.email "kinyanjuicharlesm@gmail.com"

5. Initializing a Git Repository

A Git repository is where Git stores all your file changes. To initialize a Git repository, run the following command in your project folder:

git init

6. Adding Files to the Staging Area

The staging area in Git is where changes are prepared before they are committed. To add files to the staging area, use:

git add .

Tracked files are being monitored by Git, while untracked files are new files Git hasn't started tracking yet.

7. Committing Changes

To commit your changes, use the following command:

git commit -m "Your commit message"

Commit messages should be clear and concise. To view past commits, use git log, and to revert to a previous commit, use git checkout [commit-hash].

8. Branches in Git

Branches in Git allow you to work on different parts of your project without affecting the main codebase. They are essential for collaboration and feature development.

9. Creating a Branch

To create a new branch, use the following command:

git branch new-branch

10. Viewing All Branches

To view all branches in your repository, use:

git branch

The current active branch is marked with an asterisk (*).

11. Switching Active Branches

To switch between branches, use:

git checkout new-branch

12. Merging a Branch

To merge a branch into your current branch, use:

git merge new-branch

13. Deleting a Branch

Once you’ve merged a branch and no longer need it, delete it using:

git branch -d new-branch
